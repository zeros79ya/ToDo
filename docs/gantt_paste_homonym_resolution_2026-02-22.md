# 간트 차트 근태 데이터 붙여넣기 기능 개선 내용 (2026-02-22)

## 1. 개선 요약
기존에 My HR 시스템에서 복사한 근태 데이터를 `Ctrl + Shift + V` 로 간트 차트에 붙여넣을 때 발생하던 **동명이인 일정 누락 문제** 및 **연도/단일 날짜 파싱 오류**를 해결하고, 스마트한 매칭 기능을 추가했습니다.

## 2. 주요 개선 항목

### A. 동명이인(Homonym) 수동 선택 및 자동 기억 기능
- **문제점:** '이정주' 등 동명이인이 존재할 경우 시스템이 누구의 일정인지 확신할 수 없어 간트 차트에서 일정이 아예 누락되는(그려지지 않는) 현상 발견.
- **개선 내용:**
    - 팀 멤버 이름 중복(동명이인) 판별 로직 추가 (`trip-parser.ts`)
    - 붙여넣기 시 동명이인 일정이 발견되면 **선택 팝업창 UI (TripHomonymResolverDialog)** 제공.
    - 팝업창에서 부서, 그룹, 파트, 사번(Knox ID), 연차 정보를 확인하고 올바른 주인을 수동으로 정확히 맵핑 가능.
    - **스마트 오토 리졸빙 (Auto-resolving):** 한 번 수동으로 맵핑하여 데이터베이스에 저장된 스케줄(동일 날짜, 동일 목적)은 다음 날 다시 복사해서 붙여넣어도 다시 묻지 않고 **이전 선택을 기억하여 자동으로 매칭** 처리.

### B. 스마트 연도(Year) 추출 및 단일 날짜 파싱 완벽 지원
- **문제점:** 
    1. 데이터에 기입된 날짜에 연도가 명시되지 않았을 때, 무조건 '현재 연도(수행 당일)'로 고정되어 과거 데이터 입력 시 연도 꼬임 발생.
    2. `02-19` 처럼 하루짜리(단일 날짜) 일정의 경우 파싱 정규식의 한계로 아예 일정으로 인식되지 못하고 버려짐.
- **개선 내용:**
    - 조회 기간 안내 텍스트(`YYYY-MM-DD ~ YYYY-MM-DD` 등)에서 **명시된 연도를 자동 스캔/추출**하여 해당 클립보드 데이터의 전체 기준 연도로 설정하는 지능적 파싱 로직 도입.
    - 정규식(Regex)을 업그레이드하여 기간 형식(`MM-DD ~ MM-DD`)뿐만 아니라 단일 날짜 형식(`MM-DD`)도 완벽히 인식하고 시작/종료일을 동일하게 처리하여 누락 방지.

## 3. 관련 파일 및 컴포넌트
- `src/lib/trip-parser.ts`: 모호한 일정(동명이인) 반환, 연도 추출, 단일 날짜 정규식 처리 로직 적용
- `src/components/trip-board.tsx`: 클립보드 붙여넣기 핸들러에 오토 리졸빙 로직 및 팝업 연동 구현
- `src/components/trip-homonym-resolver-dialog.tsx`: 신규 UI 컴포넌트 추가

## 4. 기대 효과
이름만 존재하는 불친절한 시스템 HR 데이터의 한계를 극복하고, 단 하나의 일정도 공중에 뜨거나 버려지지 않고 100% 완벽하게 팀원들의 간트 차트에 안착하게 됩니다.
